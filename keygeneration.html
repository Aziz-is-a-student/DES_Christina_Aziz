Parity Drop (64-bit to 56-bit)

The initial 64-bit key contains 8 parity bits (every 8th bit) used for error detection. These are dropped to create a 56-bit key using the PC-1 permutation table.

The PC-1 permutation table rearranges the remaining 56 bits while dropping the parity bits. The result is split into two 28-bit halves for the next step.
2

Round Key Generation

The 56-bit key is split into two 28-bit halves (C0 and D0). Each half is shifted left by 1 or 2 bits in each round according to the shift schedule, then combined and permuted to create the round key.

Shift Schedule

Rounds 1, 2, 9, 16: Shift by 1 bit (bits move left by 1 position)
Other rounds: Shift by 2 bits (bits move left by 2 positions)

Circular shift means the bits that fall off the left end are moved to the right end.
